# Tone of Voice

This guide defines how Aftertalk should sound across README, docs, release notes, and website copy.

## Core Style

- Direct
- Technical
- Concrete
- Confident
- Never promotional

If a sentence could fit any generic SaaS landing page, rewrite or remove it.

## Rules

- Prefer specific, verifiable claims over abstract value statements.
- Use short sentences and active voice.
- Name real components and behaviors (`WebRTC`, `STT`, `LLM`, `webhook`, `JWT`).
- Avoid hype words and vague adjectives.
- Keep language plain and precise.

## Good Examples

- "Aftertalk intercepts WebRTC audio and generates structured session minutes."
- "Zero audio stored. Minutes delivered via webhook."
- "PHP backend + TypeScript frontend, connected in 20 lines."

## Avoid

- "Revolutionary AI-powered meeting intelligence platform."
- "Seamlessly integrates with your existing workflow."
- "Enterprise-grade privacy-first solution."

## Language

- Primary language: English.
- Keep terminology consistent with the wiki and API docs.
